# Sam MultiNecklaces

BepInEx-плагин для **Cult of the Lamb**, позволяющий одному последователю носить **несколько разных
ожерелий** одновременно. Эффекты всех надетых ожерелий сохраняются; на модели показывается только одно
выбранное ожерелье, но скрытые продолжают действовать.

- GUID: `com.sam.cultofthelamb.multinecklaces`
- Версия игры, под которую собрано: Unity 2022.3.62f2, build 22885603
- Зависимости: BepInEx 5.4.x (пак для COTL), COTL_API 0.3.4 (мягкая зависимость)

## Установка
Папка устанавливается в:
```
<Папка игры>\BepInEx\plugins\SamMods\MultiNecklaces\
```
Внутри: `Sam.MultiNecklaces.dll`, конфиг создаётся при первом запуске в `BepInEx\config\com.sam.cultofthelamb.multinecklaces.cfg`.

## Как пользоваться
1. Встаньте рядом с последователем.
2. Нажмите **F8** (настраивается в конфиге, `UI/PanelHotkey`) — откроется панель «Ожерелья» для
   ближайшего последователя.
3. В разделе **«Надеть из инвентаря»** нажмите `+` у нужного ожерелья — оно надевается (списывается из
   инвентаря). Один и тот же тип нельзя надеть дважды.
4. В разделе **«Надетые ожерелья»**:
   - **Показать** — сделать ожерелье видимым на модели (эффекты остальных продолжают работать);
   - **Снять** — вернуть ожерелье в инвентарь (для редких — с подтверждением).
5. **«Подготовить сохранение к удалению мода»** — возвращает все скрытые ожерелья в инвентарь, оставляя
   у каждого последователя только видимое (выполните перед удалением мода).

## Что реализовано
- Несколько разных ожерелий на одном последователе; запрет дубликатов до списания предмета.
- Полный список надетых ожерелий с выбором видимого и снятием.
- На модели видно только одно ожерелье (ванильное поле `Necklace` = видимое ожерелье — «мост»).
- Складывание эффектов **скрытых** ожерелий: **демон-уровень (Necklace_Demonic, +2)** и
  **бессмертие (Necklace_Gold_Skull)** — через патч соответствующих методов `FollowerInfo`
  (`GetDemonLevel`, `HasTraitFromNecklace`). Видимое ожерелье считается ванилой, скрытое добавляется
  модом — двойного применения нет.
- Собственное JSON-хранилище **на каждый слот сохранения** (`saves\Sam.MultiNecklaces.slot{N}.json`):
  атомарная запись (temp → проверка → замена), версия схемы, резервная копия `.bak`, отдельная копия
  перед миграцией.
- Стабильный ключ последователя = постоянный `FollowerInfo.ID`.
- Смерть **не** снимает ожерелья; воскрешение сохраняет набор (данные привязаны к ID).
- **Разделка трупа**: после фактического завершения все оставшиеся ожерелья выпадают по одному
  (видимое выбрасывает ваниль, скрытые — мод), записи очищаются; повторный callback не дублирует
  (журнал идемпотентности).
- Русская и английская локализация (выбор по языку игры, иначе по системе, иначе английский).

## Известные ограничения (честно)
- **Эффекты скрытых ожерелий** стекаются пока только для типов с чистыми методами-геттерами
  (`Necklace_Demonic`, `Necklace_Gold_Skull`). Остальные ожерелья (сон `Necklace_5`, холод
  `Necklace_Winter/Frozen`, спасение от смерти `Necklace_Deaths_Door`, лояльность и т. д.) в v1
  действуют **только когда являются видимым** ожерельем. Причина: их проверки — это рассеянные
  сравнения поля `Necklace` в десятках мест (`FollowerBrain`, `Follower`, `RitualSacrifice` …);
  безопасно стекать их без декомпиляции каждого места нельзя. Это помечено для следующей версии.
- **Управление через панель — основной путь.** Не меняйте ожерелье у «много-ожерельного»
  последователя через ванильное меню внешнего вида: ваниль вернёт старое ожерелье в инвентарь и
  создаст рассинхрон. Используйте панель.
- Сценарии «надеть/снять/разделка/воскрешение» собраны и проверены на **компиляцию и загрузку**;
  полная игровая проверка нажатий должна выполняться вручную (см. smoke-test в отчёте проекта),
  так как автоматизация ввода в игре недоступна.

## Откат / удаление
1. Откройте панель и нажмите «Подготовить сохранение к удалению мода».
2. Запустите `uninstall_sammods.ps1` из комплекта (удаляет только наши файлы).
3. Либо вручную удалите папку `BepInEx\plugins\SamMods\MultiNecklaces\` и файл
   `saves\Sam.MultiNecklaces.slot*.json`.

Резервные копии сохранений мод делает автоматически (`saves\Sam.MultiNecklaces.vanillabackup.slot{N}`)
и хранит `.bak` рядом с основным файлом.
